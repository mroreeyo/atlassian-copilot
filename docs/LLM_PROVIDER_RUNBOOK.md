# LLM Provider Runbook

Atlassian Copilot supports personal LLM provider settings through the Broker boundary only.

## What users paste in `/settings`

- OpenAI / GPT: an OpenAI platform API key, not a ChatGPT Plus subscription.
- Claude: an Anthropic Console API key, not a Claude Pro/Max login.
- OpenRouter: an OpenRouter dashboard API key. The default model is `openrouter/auto`, or users can paste a specific OpenRouter model slug.
- Optional model name. If blank, the Broker uses a conservative default that can be changed later in Settings.

## Broker-only storage

The browser posts keys to `POST /api/settings/llm` and never stores them in localStorage, sessionStorage, frontend env, or frontend source code. The Broker encrypts personal profiles under:

```txt
~/.atlassian-copilot/broker/llm-profile.json
~/.atlassian-copilot/broker/.credential-key
```

Set `AKC_BROKER_STATE_DIR` to move that local Broker state. `DELETE /api/settings/llm` clears only the LLM profile; Atlassian settings remain intact.

## Model catalog dropdown

The Settings model selector is fed by `GET /api/settings/llm/providers/:provider/models` through the Broker only. The browser sends the provider name and optional refresh intent; it must not call OpenAI, Anthropic, or OpenRouter model APIs directly and must not receive API keys.

Catalog behavior:

- OpenAI and Anthropic live catalogs require a matching Broker-side key.
- OpenRouter catalog lookup also stays Broker-side; the Broker may use a public catalog fallback when no key exists.
- Missing credentials or provider failures return recommended defaults plus `manualEntryAllowed: true` so users can type a custom model ID.
- Cache metadata is sanitized (`hit`, `miss`, `stale`, or `disabled`) and never includes credential material.

## Runtime behavior

Provider precedence:

1. Personal `/settings` LLM provider if enabled and key-configured.
2. Broker environment fallback when explicitly enabled.
3. Mock summary stream.

Generic demo prompts may still use fictional Jira/Confluence evidence, but assigned-issue prompts can use Broker-side read-only Jira evidence. The summary segment can be generated by the configured Broker-side OpenAI, Claude, or OpenRouter adapter.

## Cost and validation

Saving a key does not call the provider. `Test connection` performs a small provider call and may incur provider cost. If no provider is configured, mock is selected, or the saved provider is disabled, the Broker returns a safe `409` response and does not call the provider or record validation state. Failed provider calls return calm status messages and do not expose keys.

After building, `npm run smoke:llm` exercises the same provider-neutral runtime path. It skips safely when no enabled provider is available.
